# Автоматическая загрузка документов из API ГосПлана

## Обзор

Реализована система автоматической загрузки документов тендеров из API ГосПлана. Система позволяет:
1. Автоматически загружать документы после поиска новых тендеров
2. Синхронизировать документы по расписанию
3. Управлять настройками загрузки через конфигурацию
4. Обрабатывать ошибки и повторные попытки

## Архитектура

### 1. Новая модель данных

**DocumentDownloadTask** - модель для управления очередью задач на загрузку документов:
- `TenderId` - ID тендера
- `DocType` - тип документа ("notification", "all", или конкретный тип)
- `Status` - статус задачи (Pending, InProgress, Completed, Failed, Cancelled)
- `Priority` - приоритет задачи (high, normal, low)
- `RetryCount` - количество попыток
- `NextRetryAt` - время следующей попытки
- `ErrorMessage` - сообщение об ошибке

### 2. Фоновые сервисы

#### DocumentSyncBackgroundService
- **Назначение**: Синхронизация документов по расписанию (каждые 2 часа)
- **Функциональность**:
  - Получает тендеры, требующие синхронизации документов
  - Создает задачи на загрузку документов
  - Обрабатывает задачи с учетом приоритетов
  - Реализует механизм повторных попыток при ошибках
  - Соблюдает rate limit API (задержка между запросами)

#### TenderSearchBackgroundService (расширен)
- **Новая функциональность**:
  - Автоматическое создание задач на загрузку документов для новых тендеров
  - Контролируется настройкой `DownloadAfterSearch`
  - Загружает только извещения или все документы в зависимости от настроек

### 3. Настройки системы

Настройки определены в классе `DocumentSyncSettings` и конфигурируются в `appsettings.json`:

```json
"DocumentSyncSettings": {
  "EnableAutoSync": true,
  "DownloadAfterSearch": true,
  "SyncIntervalHours": 2,
  "SyncOnlyCompatibleTenders": true,
  "MinMatchScore": 60,
  "ResyncExistingDocuments": false,
  "MaxTendersPerSync": 10,
  "DownloadNotification": true,
  "DownloadAllDocuments": false,
  "DocumentTypesToDownload": [],
  "ApiDelaySeconds": 2,
  "MaxRetryCount": 3,
  "RetryDelayMinutes": 5
}
```

### 4. Расширение существующих сервисов

#### DocumentService
Уже имел базовую функциональность для загрузки документов. Остался без изменений, так как уже реализовывал:
- `DownloadDocumentAsync()` - загрузка конкретного типа документа
- `DownloadAllDocumentsAsync()` - загрузка всех документов тендера
- `GetNotificationDocumentAsync()` - получение извещения

#### ApplicationDbContext
Добавлен `DbSet<DocumentDownloadTask>` для работы с очередью задач.

## Рабочий процесс

### 1. Автоматическая загрузка после поиска
```
Поиск тендеров → Сохранение тендеров → Создание задач на загрузку → Фоновая обработка задач
```

### 2. Периодическая синхронизация
```
Запуск по расписанию → Получение тендеров без документов → Создание задач → Обработка задач
```

### 3. Обработка ошибок
```
Ошибка загрузки → Увеличение счетчика попыток → Планирование повторной попытки → Макс. 3 попытки
```

## Преимущества реализации

1. **Полная автоматизация**: Документы загружаются без ручного вмешательства
2. **Гибкая настройка**: Контроль через конфигурационный файл
3. **Надежность**: Механизм повторных попыток и обработка ошибок
4. **Эффективность**: Соблюдение rate limit API, приоритетная обработка
5. **Масштабируемость**: Очередь задач позволяет обрабатывать большое количество тендеров

## Интеграция с анализом технологий

Система автоматической загрузки документов интегрирована с существующим анализом технологий:

1. **Фильтрация по совместимости**: Можно настраивать загрузку только для совместимых тендеров
2. **Фильтрация по оценке**: Загрузка только для тендеров с высокой оценкой совместимости
3. **Автоматический анализ**: После загрузки документов можно автоматически запускать анализ технологий

## Мониторинг и управление

### Логирование
Все операции логируются с различными уровнями детализации:
- Информационные сообщения о начале/завершении синхронизации
- Предупреждения при отсутствии тендеров для синхронизации
- Ошибки с детальной информацией для отладки

### Статус задач
Статусы задач хранятся в БД и могут быть использованы для:
- Мониторинга прогресса загрузки
- Анализа проблемных тендеров
- Статистики успешности загрузки
